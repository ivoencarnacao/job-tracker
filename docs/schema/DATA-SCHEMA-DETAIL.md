# Data Schema (Column-Level Detail)

This document provides the full column-level database specification, ready to drive Flyway migration scripts. For a conceptual overview, see [DATA-SCHEMA.md](DATA-SCHEMA.md).

---

## Conventions

| Convention | Value |
|-----------|-------|
| Schema | `core` |
| Primary keys | UUID, generated by application layer |
| Timestamps | `TIMESTAMPTZ` stored in UTC |
| Enums | Stored as `VARCHAR`, validated in application layer |
| Soft delete | Not used in MVP (hard delete with cascades) |
| Audit columns | `created_at`, `updated_at` on all tables |
| Null strategy | Columns are `NOT NULL` unless explicitly marked nullable |

### core.job_application_skills

Join table for the many-to-many relationship between applications and skills.

| Column | Type | Nullable | Default | Constraints |
|--------|------|----------|---------|-------------|
| job_application_id | UUID | NO | | FK → job_applications(id) ON DELETE CASCADE |
| skill_id | UUID | NO | | FK → skills(id) ON DELETE CASCADE |
| created_at | TIMESTAMPTZ | NO | now() | |

**Primary key:** Composite (job_application_id, skill_id)

## Indexes

| Table | Index | Columns | Notes |
|-------|-------|---------|-------|
| job_applications | idx_job_applications_user_id | user_id | All queries are scoped by user |
| job_applications | idx_job_applications_status | status | Dashboard counts, filtering |
| job_applications | idx_job_applications_created_at | created_at | Sorting, "this week" metrics |
| interviews | idx_interviews_job_application_id | job_application_id | Lookup by application |
| interviews | idx_interviews_scheduled_at | scheduled_at | Upcoming interviews query |
| follow_ups | idx_follow_ups_job_application_id | job_application_id | Lookup by application |
| follow_ups | idx_follow_ups_due_date | due_date | Pending/overdue follow-ups |
| follow_ups | idx_follow_ups_completed | completed | Filter pending only |
| offers | idx_offers_job_application_id | job_application_id | Lookup by application (UNIQUE already covers this) |
| job_application_skills | idx_jas_skill_id | skill_id | Skill frequency queries |
| skill_aliases | idx_skill_aliases_alias | alias | Lookup during extraction (UNIQUE already covers this) |
